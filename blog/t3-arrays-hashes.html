Array vs. Hashes
February 1, 2015

What Are Arrays and Hashes?

Essentially arrays and hashes are both indexed collections of objects such as integers, floats or strings or boolean values. Arrays are ordered lists and each of the list items can be accessed by an integer index. One thing to note is that array indexes start as 0 rather than 1. Using an integer index offers more efficiency in access the array elements but hashes offer more flexibility.

Example: There are a total of 4 elements in array

  • array = [1, "two", 3.0, true]

  • array[0] = 1
  • array[1] = "two"
  • array[2] = 3.0
  • array[3] = true

Hashes is considered more of a collection of key/value pairs rather than just a list. The key can be of any object type and must be unique while the value can be duplicated. The order of elements is not as important as arrays since hashes enumerate their values in the order that the corresponding keys were inserted.

Example: The age hash holds the age of various people

  • age = { "Matthew" => 10,
  • "Angie" => 6,
  • "Brian" => 6,
  • "Jasmine" => 9
  • }

  • age["Angie"] = 6
  • age["Jasmine"] = 9
  • age["Matthew"] = 10
  • age["Brian"] = 6

Different Applications

Arrays and hashes both store information and if the order of the elements are important then arrays may be the better option than hashes. Here I created an array called 'planets' to store a list of the planets in the solar system closest to the sun. The array will display the planets in the order the array was created.If I wanted to look up the 3rd planet from the sun I would look it up using index 2 since the index starts at 0.

  • planets = [ "Mercury", "Venus", "Earth", "Mars", "Jupiter", "Saturn","Uranus", "Neptune"]

  • puts planets
  • Mercury
  • Venus
  • Earth
  • Mars
  • Jupiter
  • Saturn
  • Uranus
  • Neptune

  • puts planets[2]
  • Earth

Here I created a hash to keep track of my friends and family's birthdays. I would store their names as keys and their birthdays as the value. Since keys must be unique, modifications would be necessary for any duplicate names. Values do not have to be unique so it will not create a problem if more than one person have the same birthday. It is far easier to retieve a birth date using the person's name rather than trying to remember the index number for each person.

  • birthdays = { "Mom" => "March 21",
  • "Dad" => "March 19",
  • "Joe" => "June 10",
  • "Joe C." => "April 11",
  • "Carol" => "June 10"
  • }

  • birthdays["Joe"] = June 10
  • birthdays["Carol"] = June 10
