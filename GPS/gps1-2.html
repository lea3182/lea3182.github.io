What is pair programming?

A technique used for learning or implementing that involves two people collaborating as either a driver or a navigator. The developers can switch roles consistently and learn from one another regardless of level.

Why is pair programming important to DBC?

Pair programming teaches you how to work with someone else. The learning process is also accelerated because you have twice the amount of brain power focusing on the same problem. Programming is a collaborative enviornment!

Compare Driver and Navigator

  1. Driver is the one who is typing while navigator is instructing.
  2. Navigator will point out syntax or logic errors whereas the driver is creating the code.
  3. Both roles contribute to making the code and both should speak up when there is a bug or a design flaw.
